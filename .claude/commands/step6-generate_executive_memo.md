# Step 6 — Executive Memo (Part 4)

Generate a 1-page C-suite executive memo from the vendor analysis and save it as `outputs/Executive Memo.docx`.

## Context

You are acting as VP of Operations for a recently acquired company (~$1B ARR). You have completed a full vendor analysis across ~400 vendors and $7.9M in annual spend and identified the top 3 savings opportunities.

Source files to read before writing:
- `outputs/top3_opportunities.md` — the three strategic opportunities with savings estimates
- `outputs/pivot_summary.txt` — department spend breakdown
- `outputs/vendors_final.csv` — full vendor categorization with recommendations

## Instructions

### Step 1 — Invoke the executive-briefing skill

Use the `executive-briefing` skill to compose the memo content. The memo must follow the BLUF (Bottom Line Up Front) format and fit on one page. Audience: **CEO and CFO**.

Structure the memo as follows:

```
═══════════════════════════════════════════════════════════
EXECUTIVE BRIEFING: Vendor Portfolio Review — Cost Reduction Opportunities
Date: [Today's date] | Prepared for: CEO & CFO | Prepared by: VP of Operations
═══════════════════════════════════════════════════════════

BOTTOM LINE
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
[MAIN SUMMARY]

KEY FINDINGS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
• [Finding 1 — top spend concentration data point]
• [Finding 2 — vendor fragmentation data point]
• [Finding 3 — duplication/overlap data point]
• [Finding 4 — total recommendation breakdown: X Terminate, Y Consolidate, Z Optimize]

TOP 3 SAVINGS OPPORTUNITIES
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
1. [Opportunity 1 title] — Est. $X savings — [1-sentence rationale]
2. [Opportunity 2 title] — Est. $X savings — [1-sentence rationale]
3. [Opportunity 3 title] — Est. $X savings — [1-sentence rationale]

IMPLEMENTATION TIMELINE
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
• Month 1–2: [Quick wins — terminations and audit initiation]
• Month 3–4: [Renegotiations and consolidation kick-offs]
• Month 5–6: [Contract exits and platform migrations]
• Month 6+:  [Full savings realization; ongoing vendor governance]

RECOMMENDED ACTIONS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
1. [Action] — Owner: VP Operations — [Timeline]
2. [Action] — Owner: CFO / Finance — [Timeline]
3. [Action] — Owner: Procurement — [Timeline]

RISKS & CONSIDERATIONS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
• [Risk 1 — e.g., vendor dependency or contractual lock-in]
• [Risk 2 — e.g., operational disruption during consolidation]
• [Risk 3 — e.g., headcount or transition cost]

═══════════════════════════════════════════════════════════
Auto Generated by AI with Confidence: [Confidence Level]
Source: Internal vendor spend data (12-month)
Reviewer: VP of Operations
═══════════════════════════════════════════════════════════
```

Fill in all bracketed placeholders using real data from the source files. Do not use placeholder text in the final output. Keep language direct, specific, and metric-driven (use "$" figures, vendor counts, percentages). No jargon.

### Step 2 — Invoke the docx skill to produce the .docx file

Use the `docx` skill (Creating New Document workflow) to export the finalized memo content as a professionally formatted Word document.

**MANDATORY**: Before writing any code, read `.claude/skills/docx/docx-js.md` in full without any line limits.

Then write a JavaScript script at `scripts/generate_executive_memo.js` that uses the `docx` npm library to produce `outputs/Executive Memo.docx`. Apply the following formatting:

- **Page size**: A4, all margins 2.54 cm (1 inch)
- **Header bar**: Full-width dark blue (#003366) shaded paragraph containing the title "EXECUTIVE BRIEFING: Vendor Portfolio Review — Cost Reduction Opportunities" in white bold 14pt Calibri, centered
- **Sub-header line**: Date, Prepared for, Prepared by — 10pt Calibri, centered, dark gray
- **Section headings** (BOTTOM LINE, KEY FINDINGS, etc.): Bold, 10pt Calibri, all-caps, preceded by a thin dark blue top border
- **Body text**: 10pt Calibri, 1.15 line spacing
- **Bullet points**: Indented, 10pt Calibri, using "•" character
- **Footer**: "CONFIDENTIAL — For internal executive use only" centered, 8pt, italic

Run the script with Node.js:
```bash
node scripts/generate_executive_memo.js
```

Verify `outputs/Executive Memo.docx` was created successfully.

## After completing both steps

1. Confirm `outputs/Executive Memo.docx` was created
2. Print the full plain-text memo content to the terminal so the user can review it
3. State the total estimated savings and the three initiative titles in one line each
